Istražite ove dvije naredbe. Što radi svaka od njih?

    iptables -t nat -A POSTROUTING --out-interface <vanjsko sučelje> -j MASQUERADE
        This command adds a rule to the nat table of the iptables firewall that 
		modifies the source IP address of outgoing packets from the network
		connected to <vanjsko sučelje> to match the IP address of the interface 
		on the gateway router that leads to the Internet. This is necessary 
		because without NAT, Internet hosts would not be able to send packets 
		back to the internal network since their source IP addresses would be 
		invalid. The MASQUERADE target is used to dynamically set the source 
		IP address based on the outgoing interface, which simplifies configuration
		and allows for changes in the network topology.
    iptables -A FORWARD --in-interface <unutarnje sučelje> -j ACCEPT
        This command adds a rule to the filter table of the iptables firewall that allow
		s incoming packets on the <unutarnje sučelje> interface to be forwarded to their
		destination. The ACCEPT target is used to allow the traffic, whereas DROP 
		target would reject it. This rule is necessary to allow traffic to flow
		between the internal network and the Internet, since by default the Linux 
		kernel blocks forwarding of packets between different network interfaces.
